# Design Philosophy

Build products, not pages.

Create experiences, not CRUD screens.

Every interface must feel:

* Modern
* Premium
* Elegant
* Professional
* Polished
* Production-ready

The result should resemble software used by real companies.

Never create interfaces that feel like student projects.

# UI Standards

Always prioritize:

1. Visual hierarchy
2. User productivity
3. Readability
4. Accessibility
5. Responsiveness
6. Consistency
7. Reusability

Every screen must have:

* Clear hierarchy
* Strong spacing system
* Modern typography
* Consistent components
* Meaningful interactions

---

# Layout Principles

Prefer:

* Sidebar navigation
* Clean top navigation
* Dashboard layouts
* Modular content sections
* Responsive grids
* Card-based organization

Avoid:

* Crowded layouts
* Giant blocks of text
* Inconsistent spacing
* Generic Bootstrap styling

---

# Spacing Rules

Use generous spacing.

Prefer breathing room over density.

Recommended spacing scale:

* 4px
* 8px
* 12px
* 16px
* 24px
* 32px
* 48px
* 64px
* 96px

Maintain consistent spacing throughout the application.

---

# Typography Rules

Use modern typography hierarchy.

Prioritize readability.

Headings should be bold and confident.

Body text should be comfortable to read.

Avoid tiny fonts.

Use clear distinction between:

* Headings
* Subheadings
* Body text
* Captions

---

# Color Philosophy

Use color intentionally.

Avoid excessive saturation.

Prefer:

* Neutral surfaces
* Strategic accents
* Soft gradients
* Subtle depth

Avoid rainbow color schemes.

---

# Cards

Cards should feel premium.

Requirements:

* Large border radius
* Soft shadows
* Clear hierarchy
* Proper padding
* Hover feedback

Cards should never feel flat or outdated.

---

# Tables

Tables must feel modern.

Requirements:

* Sticky headers
* Zebra rows
* Hover states
* Search tools
* Filter controls
* Status badges
* Clean pagination

Avoid plain HTML table appearance.

---

# Forms

Forms must be easy to scan.

Requirements:

* Proper grouping
* Clear labels
* Inline validation
* Helpful descriptions
* Consistent spacing

Users should immediately understand what to do.

---

# Buttons

Buttons should communicate priority.

Primary actions:

* Strong visual emphasis

Secondary actions:

* Lower visual weight

Always include:

* Hover state
* Focus state
* Disabled state
* Loading state

---

# Animations

Use animations carefully.

Preferred:

* Fade
* Slide
* Scale
* Micro interactions

Animation duration:

150ms–300ms

Avoid distracting effects.

---

# Empty States

Never leave empty screens.

Create useful empty states with:

* Illustration
* Explanation
* Suggested action

---

# Dashboard Standards

Every dashboard should include:

* Welcome section
* KPI cards
* Charts
* Recent activity
* Quick actions
* Insights

Dashboards should feel alive.

Avoid empty white screens.

---

# Landing Page Standards

Landing pages should include:

1. Hero Section
2. Features Section
3. About Section
4. Statistics Section
5. Testimonials
6. FAQ
7. Final CTA
8. Footer

The landing page should tell a story.

---

# Accessibility

Follow WCAG AA.

Ensure:

* Keyboard navigation
* Focus visibility
* Color contrast
* Semantic HTML
* Screen reader compatibility

Accessibility is mandatory.

---

# Code Standards

Prefer:

* Reusable components
* Clean architecture
* Maintainable code
* Consistent naming
* Scalable structure

Avoid:

* Duplicate code
* Hardcoded styles
* Monolithic components
